ERYA – Profiling Calculator



This widget is intended both to use as a generic calculator, that resemble the ones bundled on several operating systems, and also to serve mainly as a debug tool to the Detector Profile and the Detector and Ressonance tab, specially if the user suspects of any abnormal calculation.


Quick Reference


To use the built-in Calculator, input the algebraic expression (or use the keyboard, containing all the reserved words as functions, and special variables) on the first input box, and then hit the “Return” button to display the result, or an error, on the second box.

The ERYA calculator uses the same macro language shared along several program components, making possible to serve as a debug tool. Any crossed reference about the macro language will apply on this Calculator applet.


Numerical Calculations

When the Calculator returns an expected numerical value, the utility is on standard mode, and save the last numerical result on the memory. When click on “DspAns”, it will clear the input box, and display the recorded result on scientific notation.

Then, just add another sequence of commands, once use the colon “:” first.

Example:

Ans = 0.0000000000e+00 [Add some commands]

Ans = 0.0000000000e+00 : 2 + Ans + (Ans + 1)^3 [Hit return]

3.000000 [On Output Box]


It is possible to define constants, and use them to simplify the use of the numerical calculations:

Example:

a0 = -2.2 , a1 = 4.98 , a2 = 7.32 , a3 = -0.14 , x = 5 : a0 + a1*x + a2*x^2 + a3*x^3

188.200000


The buttons “ClrProg” clears the input box, and “ClrAns” clears the output box, respectively.


Program Calculations


The second operation of this tool is the function mode, when the user define a programmable function on the input box, similar to the Detector Profile’s Custom function and the Detector and Ressonance’s Custom Resonance function. In the mode, the Calculator can be used to check the results for several values from each independent variable.


Even so, for each evaluated function dependent variable, the calculator store the result on the memory, and can be retrieved by the “DspProg” program, returning the value on scientific notation:

Example:

FxAns = 0.0000000000e+00


A program on ERYA-Profiling built-in algebraic parser, should be declared on the first part, by defining variables, assigned to several reserved keywords called parameters.

At least, it is expected to define the dependent and independent variable of the custom function:


Example 1:

fxvar = x , fyvar = y

[Tip: Use the Calculator keys to avoid typesetting of the reserved words.]


But it is possible to define two additional variables for the function domain range:

Example 2:

fxvar = x , fyvar = y , fxmin = fa , fxmax = fb

[Note: Only the Resonance tab box requires to use the last keywords, since it was mandatory.]


From the Ziegler’s Parameters widget, the custom algebraic function also requires another kind of parameter, where it creates a fixed vector of 16 elements, where the components are named by the initial name, and the another ones are referenced by the initial variable names appended with a number:

Example 3:

fxvar = x , fyvar = y , fxmin = fa , fxmax = fb, fnvar = k

[Warning: This is intended to ERYA read an entire 16 elements line from Ziegler’s Table, not to the user replace manually.]


To complete the program, add after a colon “:” any sequence of constants. If you define the parameters of limited domain range, it is the right place to define them, or it will use the default values.

Then the two examples can be like this:

Example 1:

fxvar = x , fyvar = y : a = 10 , b = 3


Example 2:

fxvar = x , fyvar = y , fxmin = fa , fxmax = fb : a = 10 , b=3 , fa = 5 , fb = 9


Finally, once written the constants section, write the final algebraic expression on the last section, separated by a colon “:”

Example 1:

fxvar = x , fyvar = y : a = 10 , b = 3 : y = a / (x – b)


Example 2:

fxvar = x , fyvar = y , fxmin = fa , fxmax = fb : a = 10 , b=3 , fa = 5 , fb = 9 : y = a / (x – b)


Now the program are complete, and when hit the “Return” button it will get a “0.000000” result, unless you make a mistake, and return an error instead.


To use the Calculator on the program mode, use the “f(x=?)” button instead, and it will ask to input a numerical value on a separate dialog box. Once click “OK” to validate, the answer should display on the output box.

If you use the “DspProg”, it will display the answer on scientific notation, but it will clear the input box, erasing the program.

If you define a finite domain range, any input outside the function domain will get always a “0” as answer.
