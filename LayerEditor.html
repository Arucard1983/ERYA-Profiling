ERYA Profiling : Layer Profiler Editor


The ERYA Profiling requires a precise definition of the sample contents, and the “Layers” tab on the Main Screen are the most critical step to achieve this task.


The main spreadsheet is easily editable, and should be take attention that the columns contents are generated by the previous “Elements” tab, and change dynamically as long their contents are altered.


Some Basic Concepts


A sample is a collection of several layers, representing a peculiar slice of sample with a local homogeneity. A stack of homogeneity layers forms the entire sample, which is the model of inhomogeneity sample implemented on ERYA-Profiling

Each layer shall contains a particular description of each Element quantities. Natively, all layer contents are stored on a superficial atomic density (number of atoms per square centimeter, more precisely, on 1015 atoms units per square centimeter), and for each Element name, the user should insert the relative stoichiometry.

It is not absolutely necessary to insert a normalized stoichiometry, as the program make it automatically, when the simulation main procedure starts.


How to define the number of Layers on the Sample


The number of rows on Layer Editor spreadsheet are dynamic, where each one should represent a physical layer, and the table’s number of rows can be changed by the “Number of Layers” switch.

When adding or removing some layers, the untouched rows contents will not be erased.


Load and Save a Layer Profiler


It is highly recommendable to store any experimental sample composition to a file, avoiding the time consuming work to fill manually all the data each time you want to compute the same sample for every experience, even it changes a little.

It is highly advantageous to save and load any sample composition on their native XML format, which have an epls (ERYA Profiling Layer Sample) extension, even it can only be edited by this program.

Support for plain, but column tabbed, ASCII text file are implemented, but warns the user that this format is deprecated, since lacks some control features.

The main reason is the fact that the previous LabView ERYA programs reads samples on text format, but do not support labeled columns.

In this case, ERYA-Profiling will imitate that behavior (while ignore non-numerical lines), to provide a quick transition between the two softwares.

Even it was an additional drawback that are the necessity to use the “Elements” tab to define all the Elements forehand.


Using the native sample profile file, the element names are placed as labels, are makes possible for the program to check if the Element Names exists, or warns the user to check the default Database, and aborts the profile import.

Additionally, the “Elements” tab are automatically created, without any user previous input.

If the file import aborts with an error, the user should check if the database are sane, or contains the non-existent Element, otherwise you should load a different Database before continue.


Finally, a third option is to save and load the sample on an Excel Xlsx file, which turns possible to define samples, or edit them by other programs, like Google Docs®, LibreOffice®, Microsoft Office®, and others, as long they support writing to Microsoft Excel 2007 Xlsx file format, at least by a basic level.


Another benefit to use the Excel option, are the possibility to use a first row with labels, helping ERYA to check the sample consistency, and even to verify if the Element exists on the database.

This makes possible to combine the benefits of the native epls sample file, with a more generic spreadsheet software.


A Compatible Sample created by an Excel Xlsx File


It requires the following requirements:


1. Save all data on the first worksheet, on Excel 2007 Xlsx format.


2. All data should be packed together like a single closed boundary matrix, or the ERYA Xlsx file filter may gives wrong data extraction. There’s no need to place the first relevant cell on the first address of the spreadsheet, since during the decoding, it will trim the matrix references.


3. The first row should contains only labels, where the first column should place any name string.

The latter columns, should be placed with the Element name, along the Gamma Peak energy.

If the Element don’t have any Gamma Peak (placeholder Elements), you can omit this reference.


4. The label columns should respect the following formats, while notice that the contents between the bold square brackets are examples, and should omit the brackets ones when typing:


4.1 <Element> : <Gamma> [19F : 197]

4.2 <Element> ; <Gamma> [19F ; 197]

4.3 <Element> (<Gamma>) [19F (197)]

4.4 <Element> (<Gamma> keV ) [19F (197 keV) ]

4.5 <Element> <Gamma> [19F 197]

4.6 <Element> [16O]


5. ERYA recognize the parenthesis, colons, semicolons and empty spaces as remark elements, removing them once parsed, in order to return the first two words as the Element Name and their Gamma Peak value.

The real issue is when the <Element> name and <Gamma> value do not exists, that will give a fatal error dialog, and aborting the procedure.

To solve the error, the user should check if the database are sane, or load a new one, or simply verify if you make a typo on source Excel file, and change it.

If you omit the <Gamma> field, then the program will check if the Element is a placeholder element, without gammas, and if happens the parsing are validated and resumes.


6. Once settled the first row, place the numerical stoichiometry/atomic density, according to the respective column label.

Once done, save the Excel file, and import the content using the ERYA-Profiling itself.


7. It is possible to create a pure numerical Excel file, like the ASCII text format, but the user should define forehand all Elements and their Gammas manually, before import the Excel File.

This option, albeit possible, is not recommendable.
