ERYA – Profiling Calculator



This widget is intended both to use as a generic calculator, that resemble the ones bundled on several operating systems, and also to serve mainly as a debug tool to the Detector Profile or the custom function on Detector and Ressonance tab, that uses the ERYA Macro expressions.


Quick Reference


To use the built-in Calculator, input the algebraic expression (or use the keyboard buttons to fill the functions and variables) on the first input box, and then hit the “Return” button to display the result, or an error message, on the second box.

The ERYA calculator uses the ERYA Macro language shared along several program components, to return a result, making possible to serve as a debug tool. Read here to a more complete reference.


ERYA Macro interpreter store on a temporary answer memory the last numerical result giving by an arithmetic expression, when the interpreter reads the colon (:) keyboard, or finishes the whole code sequence.

A pure arithmetic expression don’t had any variable assignment, instead when happens, the answer memory are reset to zero, and the arithmetic expression assigned to a variable will store the result on the memory stack.

To display a variable content, write the variable name after the colon (:) in the end of your code.


Numerical Calculations

To use ERYA Calculator as a normal calculator, named calculator mode, write an algebraic or arithmetic expression using the syntax rules, and press “Return” to display the value stored on answer memory.


It may be useful and more user-friendly to define constants, and write the arithmetic expression using those constants. The user can use “InpVar” button to check if a variable name are syntactical correct:

Example:

a0 = -2.2 , a1 = 4.98 , a2 = 7.32 , a3 = -0.14 , x = 5 : a0 + a1*x + a2*x^2 + a3*x^3 [Hit Return]

188.200000 [On Output Box]


Note: To display all variable contents from memory stack, click “DspVars” button to display a list of variables and their values straight on the output box.


ERYA uses the double precision arithmetic to handle all numerical calculations, and the ERYA Macro interpreter follows this standard. To display a different number of decimal places (between 6 to 15), click “SetDec” button and select the desired precision.

The buttons “ClrProg” clears the input box, and “ClrAns” clears the output box, respectively.


Program Calculations

The second operation of this tool is the function mode, when the user define a programmable function on the input box, similar to the Detector Profile’s Custom function and the Detector and Ressonance’s Custom Resonance function. Using this special feature, the Calculator can be used to check the results for several values from each independent variable.


A program on ERYA-Profiling built-in algebraic parser, should be declared on the first part, by defining variables, assigned to several reserved keywords called parameters.

At least, it is expected to define the dependent and independent variable of the custom function:

Example 1:

fxvar = x , fyvar = y

[Tip: Use the Calculator keys to avoid typesetting of the reserved words.]


But it is possible to define two additional variables for the function domain range:

Example 2:

fxvar = x , fyvar = y , fxmin = fa , fxmax = fb

[Note: Those variables should be initialized, like other variables.]


To support the Ziegler’s Parameters widget, the custom algebraic function also requires another kind of parameter, in order to copy a line from the Ziegler’s Table to form a fixed vector of 16 elements.

Due to the language limitations, the initial term are referenced by the initial name, and the another ones are referenced by the initial variable names appended with a number:

Example 3:

fxvar = x, fyvar = y, fxmin = fa, fxmax = fb, fnvar = k : y = x^k/(k1+k2*x)

[Tip: Use the “InpVec” button to override the 16 vector elements values. It will display a dialog box to the user input up to 16 numbers separated by commas, and it will be stored on the calculator memory.]


To complete the program, add after a colon “:” any sequence of constants. If you define the parameters of limited domain range, it is the right place to define them, or it will use the default values.

Example 4:

fxvar = x , fyvar = y : a = 10 , b = 3


Example 5:

fxvar = x , fyvar = y , fxmin = fa , fxmax = fb : a = 10 , b=3 , fa = 5 , fb = 9


Finally, once written the constants section, write the final algebraic expression on the last section, separated by a colon “:”

Example 6:

fxvar = x , fyvar = y : a = 10 , b = 3 : y = a / (x – b)


Example 7:

fxvar = x , fyvar = y , fxmin = fa , fxmax = fb : a = 10 , b=3 , fa = 5 , fb = 9 : y = a / (x – b)


Now the program are complete, and when hit the “Return” button it will get a “0.000000” result, unless you make a mistake, and return an error message instead.


To use the Calculator on the program mode, use the “f(x=?)” button instead, and it will ask to input a numerical value on a separate dialog box. Once click “OK” to validate, the answer should display on the output box.

If you define a finite domain range previously, any input outside the function domain will get always a “0” as answer.
