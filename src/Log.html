ERYA Profiling : Handling Log



When ERYA Profiling finishes the numerical simulation, the results are plotted on “Output” tab, and a numerical table are stored on “Table” tab. A full detailed partial calculations are also stored on the “Log” table (this one). If the user opens the “Advanced” button to disable logging option, this tab is never filled with any data.


This (usually long) table store all partial progress of yields numerical evaluation, for each relevant step, including the parameters derived from the Vavilov Distribution, partial yield results, and energy loss.

The main reason is to serve as a diagnostic tool to help the user set the most accurate and fast precision possible, before using a full blown numerical simulation.


The evaluation explanation of energy loss, distributions and partial yields can be explained from the main theory page. Leaving this section to add some notes.


The log table also evaluate two asymptotic variances (actually, more the standard-deviation, which is the square root of the variance that are displayed on the table) for the Straggling Distribution when applied to the Vavilov Distribution at the Gaussian approximation:

- The first one are the Bohr variance, evaluated from the sample chemical composition (as explained on Theory Section page), and defined by:

σB2 = (8 ξ) / (3 Γ) ∑ si Ii ln ( 2 me β2 / Ii )

- The second one, is the asymptotic Vavilov Variance, defined by:

σV2 = ξ2 (1 – β2/2)/k

(Where they are the Vavilov-Landau Parameters described here, and on theory section.)


In practice, the two variances are in the same order of magnitude, and rarely the ratio between the two are bigger that two, much due to the fact they depend from the same physical parameters of the sample, making them somehow comparable between then.


Saving the Log Table to a File


It is possible to save the table to an ASCII file or an Excel Xlsx file, using the button “Save Log Data”, but for long log tables, it can make huge Excel files, that could exceed the file format conventions when opened by the original Excel software. ERYA Xlsx file filter was not intended to create such giant files with hundreds of thousands of rows.

Using a text file instead is the most safe option on such cases.
