ERYA Profiling : Advanced Program Settings


This applet is intended to manage the ERYA-Profiling configuration file directly, while change the default databases that loads at program start-up.


How To Change the Configuration File


The default Databases files and the configuration files are always stored on the default operating system local profile directory (recommended setting), or on the same program root directory(not recommended).


The user can switch the profile mode (user or root mode folder), and select the three major database files (it will start on the same directory of the configuration setup file). The user can select any source file from every possible directory anyways.


Once all things set, click “Save”, and accept the warning dialog. ERYA-Profiling will check the new files, and copy the contents to the same directory where the configuration file exists.

In case of successfulness, the ERYA-Profiling will reload the new databases, and start working without need to restart the application.


Troubleshooting


In case of any error, ERYA-Profiling may need to restart, and if necessary, use this tool to click on “Delete”, or use “Settings”>“Reset Settings” from the main window menu, to rebuild a new configuration file from scratch.

The recommend profile setting (as user profile) is due to maintain the original Databases on the root program directory, in order to make a successful setup reset in case of a severe mistake, instead to re-install all the program again.


To Abort any Edition


To avoid overwriting the contents of current setup file, just click “Quit” to exit this applet without touching the current configuration.

The “Reset” button only clean the applet editor, and it won’t make any changes at all when click on “Quit” button.

“Delete” will only delete the configuration file, in order to force ERYA restart and opens the Configuration Wizard again.
